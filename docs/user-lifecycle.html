Patient Journey — Complete Flow

Every decision point a patient encounters from login to order completion.

01

Login / Register

Patient signs in with email+password or Google OAuth. Redirected to /patient/dashboard.

JWT issuedRole: patient
02

Create Medicine Request

Navigates to New Request. Adds medicines, optional prescription image, selects governorate, optionally shares current GPS location, chooses delivery type and payment method.

Geolocation APIPrescription uploadPayment required
03

Backend broadcasts to pharmacies

Backend finds all verified pharmacies in the same governorate and emits pharmacy:new-order via Socket.io + push notification to each.

Socket.ioNotification service
04

Receive & Compare Offers

Real-time notification arrives. Patient opens order detail page to see all pharmacy offers with prices, delivery fees, estimated times, and pharmacy ratings.

Real-time updateCompare offers
05

Confirm Offer

Patient selects one pharmacy offer. All other offers are automatically rejected. Order status moves to confirmed.

Other offers rejectedStatus: confirmed
06

Track Order in Real-time

Live status updates: Preparing → Out for Delivery → Delivered. Each update triggers a push notification.

Live trackingPush notifications
07

Receive Order & Pay

Cash: pay delivery person directly. InstaPay: transfer via mobile wallet using instructions sent through chat.

Cash on deliveryInstaPay
08

Leave Review

Rate the pharmacy (1–5 stars) with optional comment. Updates pharmacy's average rating on the platform.

Star ratingUpdates pharmacy.rating

Pharmacy Journey — Complete Flow

From initial registration through verification to daily order operations.

01

Register as Pharmacy

Fills registration form with pharmacy name, address, governorate, working hours. Account created with isVerified: false.

isVerified: falseAwaiting approval
02

Admin Verifies Account

Admin reviews pharmacy details and approves. Pharmacy receives notification: "Your account has been verified".

isVerified: truePush notification
03

Set Up Profile

Uploads working hours, sets pharmacy location, manages inventory of available medicines and prices.

Location setInventory management
04

Receive New Order Alerts

When a patient in the same governorate creates an order, pharmacy gets a real-time Socket.io event and push notification.

Socket.io eventpharmacy:new-order
05

Submit Price Offer

Reviews the order, checks inventory, and submits an offer with available medicines, alternatives, total price, delivery fee, and estimated time.

POST /api/orders/:id/responsesOffer expires if not confirmed
06

Order Confirmed by Patient

Real-time notification that the patient accepted the offer. All other pharmacy offers for this order are rejected.

order:confirmed eventStatus: confirmed
07

Update Order Status

Moves order through: Preparing → Out for Delivery → Delivered. Each step notifies the patient in real-time.

PUT /api/orders/:id/status3 transitions
08

Receive Rating

After delivery, patient leaves a star review. Rating is averaged into pharmacy's public profile score.

Affects pharmacy ranking
