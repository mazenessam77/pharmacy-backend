5. Order Lifecycle (State Machine)

Complete order status transitions from creation to delivery

stateDiagram-v2
    [*] --> pending : Patient creates order

    pending --> offered : Pharmacy submits offer
    pending --> cancelled : Patient cancels

    offered --> offered : More pharmacies respond
    offered --> confirmed : Patient accepts an offer
    offered --> cancelled : Patient cancels

    confirmed --> preparing : Pharmacy starts preparation
    confirmed --> cancelled : Patient cancels

    preparing --> out_for_delivery : Pharmacy dispatches

    out_for_delivery --> delivered : Order reaches patient

    delivered --> [*] : Complete ✓
    cancelled --> [*] : Terminated ✗

    note right of pending
        Nearby pharmacies notified
        via Socket.IO + FCM Push
    end note

    note right of offered
        Patient compares offers:
        price, distance, ETA
    end note

    note right of confirmed
        Other offers auto-rejected
        Confirmation email sent
    end note

    note left of delivered
        Patient can now
        leave a review (1-5 stars)
    end note
